# Show/Save JSON — Guide

## Назначение
Узел-постпроцесс: аккуратно отображает JSON в UI и (опционально) сохраняет в файл/директорию. Удобно для вывода transform_json, параметров нод, логов.

## Быстрый старт
1) Подайте `json_text` (строка, объект или список).  
2) (Опц.) `output_path`: файл или директория (если директория — создаст `transform.json`).  
3) Получите `json_pretty` и UI-текст.

## Поведение
- Автоматически парсит строки: `" {...}"`, `"{\"a\":1}"`, списки — приводит к pretty JSON с отступами.  
- Если передан объект/список напрямую — форматирует его.  
- Если `output_path` указан: создаёт директории при необходимости, пишет файл в UTF-8.

## Полезные приёмы
- Можно скормить выход `transform_json` из других нод — увидите читаемый JSON и получите файл.  
- Для отладки: соедините несколько JSON строк в список — узел выровняет и покажет массив.  
- Чтобы сохранить рядом с результатами, передайте путь директории (например, `output/align/`).

## Ограничения
- Не валидирует схему; выводит «как есть».  
- Если строка не парсится как JSON, покажет исходную строку внутри `"raw"`.
